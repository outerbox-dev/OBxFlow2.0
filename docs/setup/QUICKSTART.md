## Step 1: Clone Repository

```bash
git clone https://github.com/outerbox-dev/OBxFlow2.0.git
cd OBxFlow2.0/n8n
```

---

## Step 2: Run Setup Script

The automated setup script will:
- Generate secure credentials
- Create your `.env` file
- Validate configuration
- Optionally start services

```bash
./scripts/setup.sh
```

**Follow the prompts!** The script will:
1. Check prerequisites
2. Generate encryption key
3. Generate secure passwords
4. Create `.env` file
5. Ask if you want to start services now

**üíæ Save your credentials!** The script displays them once - store in a password manager.

---

## Step 3: Access N8N

If you started services during setup, N8N is ready at:

```
http://localhost:5678
```

**Login:**
- Username: `admin`
- Password: (shown during setup)

---

## Manual Start (If Needed)

If you didn't auto-start during setup:

```bash
docker-compose up -d
```

View logs:
```bash
docker-compose logs -f
```

---

## Creating Your First Workflow

1. **Login to N8N** at http://localhost:5678

2. **Click "New Workflow"**

3. **Add a Trigger Node**
   - Click the "+" button
   - Search for "Webhook"
   - Configure the webhook

4. **Add Action Nodes**
   - Connect nodes by dragging
   - Configure each node
   - Test as you go

5. **Activate Workflow**
   - Click "Active" toggle in top right
   - Your workflow is now live!

## Common Commands

```bash
# Start services
docker-compose up -d

# Stop services
docker-compose down

# View logs
docker-compose logs -f

# Restart services
docker-compose restart

# Check status
docker-compose ps

# Create backup
./scripts/backup.sh

# Restore backup
./scripts/restore.sh backup_name
```

## Troubleshooting

**Can't access http://localhost:5678**
```bash
# Check services are running
docker-compose ps

# Check logs
docker-compose logs n8n
```

**Services won't start**
```bash
# Check Docker is running
docker ps

# Recreate services
docker-compose down
docker-compose up -d
```

**Lost credentials**
```bash
# Check .env file
cat .env | grep PASSWORD
```
